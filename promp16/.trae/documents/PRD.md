# 运营反馈系统 PRD

## 1. 产品概述
一个面向运营人员与技术团队的问题反馈与值班响应平台。运营人员可快速提交问题（含紧急程度、所属类别、截图/视频等），系统根据值班安排自动指派当前值班人，紧急问题触发电话通知。所有问题支持跟进、留言、归档（带分类）。
- 目标用户：运营人员、前端/后端/安卓/iOS/测试技术值班人员
- 核心价值：缩短问题反馈到响应的链路，明确责任人，沉淀问题知识库

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 运营人员 | 系统录入 | 提交问题、留言、查看自己提交的问题 |
| 技术值班人员 | 系统录入 | 查看问题、跟进、留言、归档、维护值班信息 |
| 管理员 | 系统录入 | 管理值班组、值班人员、联系人、查看全部问题 |

### 2.2 功能模块
1. **问题列表页**：问题列表筛选、查看详情、快速跟进
2. **问题提交页**：表单录入问题信息（紧急程度、分类、描述、附件）、自动指派值班人
3. **问题详情页**：处理记录时间线、留言、跟进按钮、归档操作
4. **值班安排页**：查看每日值班表、维护值班顺序
5. **联系人管理页**：管理联系人、值班组、值班人员（含岗位）

| 页面名称 | 模块名称 | 功能描述 |

## 3. 核心流程

### 3.1 问题提交流程
运营人员进入"提交问题"页 → 填写表单（紧急程度、描述、附件、所属分类）→ 系统根据所属分类自动匹配默认值班组和当前值班人 → 可手动调整值班组或添加联系人 → 提交 → 若为紧急问题，系统触发电话通知当前值班人 → 进入问题详情页

```mermaid
flowchart TD
    "运营进入提交页" --> "填写问题表单"
    "填写问题表单" --> "系统按分类匹配值班组"
    "系统按分类匹配值班组" --> "可手动调整/添加联系人"
    "可手动调整/添加联系人" --> "提交问题"
    "提交问题" --> "紧急？"
    "紧急？" --> "是" --> "电话通知值班人"
    "紧急？" --> "否" --> "通知值班人(站内)"
    "电话通知值班人" --> "进入问题详情"
    "通知值班人(站内)" --> "进入问题详情"
```

### 3.2 值班人员分配流程
管理员在联系人管理中添加值班人并选择岗位 → 系统按岗位分组（安卓5人、iOS5人、前端10人、服务端16人、测试6人）→ 按日轮转：每个岗位每天恰好1人值班 → 生成值班表 → 问题提交时自动将当前值班人设为默认值班人

### 3.3 问题跟进与归档流程
任意技术人员进入问题详情 → 点击"跟进"按钮 → 处理记录追加"xx跟进了该问题" → 可在留言区留言 → 问题解决后点击"归档" → 选择归档分类（历史问题/已知问题/前端问题/服务端问题/操作问题/非bug）→ 问题状态变更为已归档

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深蓝 #1e3a8a （稳重、专业）
- 辅助色：紧急红 #dc2626、进行中橙 #f59e0b、已归档绿 #16a34a
- 按钮风格：圆角 6px，主色填充
- 字体：Noto Sans SC 中文显示字体，标题 18-20px，正文 14px
- 布局：左侧导航 + 顶部状态条 + 主内容区（卡片式）
- 图标：线性图标（Lucide / Heroicons 风格）

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 问题列表页 | 问题卡片 | 左侧色条表示紧急程度（红=紧急，灰=普通），标题、标签、值班人头像、状态徽章、时间 |
| 问题提交页 | 表单 | 分组卡片：基本信息 / 附件 / 值班指派，底部吸附提交按钮 |
| 问题详情页 | 时间线 | 垂直时间线展示处理记录，右侧为操作区 |
| 值班安排页 | 值班表 | 横向日期滚动 + 纵向岗位，单元格展示值班人头像+姓名 |
| 联系人管理页 | 列表 | 表格展示，行内编辑 |

### 4.3 响应式
- 桌面优先设计，最小支持 1280px 宽度
- 侧边栏在 1024px 以下收起为抽屉
- 表格在 768px 以下转为卡片列表

## 5. 非功能需求
- 数据持久化：使用 localStorage / mock 数据，支持后续接入真实后端
- 附件支持：图片（jpg/png/gif/webp）与视频（mp4/webm），单文件最大 50MB
- 电话通知：紧急问题触发模拟电话通知（toast 提示 + 模拟拨号界面）
- 数据规模：支持 1000+ 问题、100+ 联系人流畅加载
